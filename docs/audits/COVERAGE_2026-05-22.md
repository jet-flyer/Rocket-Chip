# Host C++ Line Coverage — 2026-05-22

**Tool:** `llvm-cov` (Clang 19.1.7) against `build_host_cov/` with `-fprofile-instr-generate -fcoverage-mapping -O0`.
**Test corpus:** 44 `test_*.exe` gtest binaries (856/856 PASS).
**Baseline:** `logs/audit-2026-05-13-L2/04_coverage_llvmcov.txt` (the L2 audit run, 9 days ago).

---

## Methodology

1. Rebuild `build_host_cov/` with current sources (post baro_kf + qep_smoke + Thornton removals this session).
2. Run each `test_*.exe` with `LLVM_PROFILE_FILE=test/<name>.profraw` to capture per-test profile data.
3. `llvm-profdata merge -sparse` into `coverage.profdata`.
4. `llvm-cov report` aggregated across all 44 test binaries.
5. Filter to `src/*` files only (matches the L2 baseline shape).

---

### ud_factor.cpp big jump explained

The 2026-05-13 baseline's 29.59% reflected three Thornton WMGS variants (`thornton_f32` / `thornton_mixed` / `thornton_f64`) + helpers (`ud_from_diagonal`, `ud_all_positive`) sitting in the TU with **zero production callers** and **zero TEST() blocks exercising them**. CHANGELOG 2026 entry "UD factorization + DCP float64 benchmark" had concluded "UD not needed for numerical stability; Thornton f32 predict is 29.6× slower than codegen" — the variants were Phase-1 benchmark code, kept after the investigation closed.

Per the BaroKF-class history-trace-before-deletion rule (project policy 2026-05-22, `standards/CODING_STANDARDS.md`), the Thornton variants + 2 unused helpers were removed this session. Remaining functions:
- `ud_factorize` — eskf.cpp:649 (Bierman update path)
- `ud_to_dense` — eskf.cpp:616 (P reconstruction)
- `bierman_scalar_update` — eskf.cpp:669 (measurement update inner loop)

All three are production-used and exercised by `test_eskf_bierman.cpp`. -191 lines from `src/fusion/ud_factor.cpp`.

## Reproduction

```bash
# Rebuild + collect profiles
cmake --build build_host_cov
cd build_host_cov
rm -f test/*.profraw
for exe in test/*.exe; do
    name=$(basename "$exe" .exe)
    LLVM_PROFILE_FILE="test/${name}.profraw" "$exe" > /dev/null 2>&1
done

# Merge + report
/c/Program\ Files/LLVM/bin/llvm-profdata.exe merge -sparse test/*.profraw -o coverage.profdata
OBJ_ARGS=$(ls test/*.exe | awk '{printf "-object %s ", $0}')
/c/Program\ Files/LLVM/bin/llvm-cov.exe report $OBJ_ARGS -instr-profile=coverage.profdata | grep '^src'
```
